Arch linux

Arch Linux is a 64bit OS

* This is entirely an bare bone installation everything has to be installed manually

* this is a very Light-Weight Linux Distro but Hard to install

* Arch Linux has the best user repository, this is the only distro maintained by the community

* Arch Linuux is also know Bleeding-Edge Distro because it keeps on updating almost Everyday

* And it has the one of the best package Manager pacman

* the link to the iso https://www.archlinux.org/download/ select the iso according to the country

* First load the iso in usb which will be around 600mb

* shutdown the pc and boot into bios by pressing F12 or so..

* once loading into the UEFI-boot using the iso select Boot Arch Linux (x86_64) option

* Now you will be prompted to the terminal for the installation

Installation:-

setup the disk partitions:

* fdisk -l
to list the available disks in the pc

* fdisk /dev/diskname
selecting the disk , change the disk name for eg. "fdisk /dev/sda" is my disk name

Here there are two options if you want single boot or dual boot

* g
for single boot this will create a New gpt label in disk if you want dual boot skip this step

for efi partition:

* n
this will create the new partion

* press enter
if you want fdisk to give a default partion number

* press enter
this will create label on the partion

* +512M
this is efi partition size which we use for booting the arch

* t
for selecting the type of partion

* L
for listing the types of partition names use "q" to quit the list

* 1
this for making the partition type to efi

for swap partition:

* n
this will create the new partion

* press enter
if you want fdisk to give a default partion number

* press enter
this will create label on the partion

* +2G
this is swap partition size is always depends on your ram

* t
for selecting the type of partion

* L
for listing the types of partition names use "q" to quit the list

* 19
this for making the partition type to linux-swap

for mnt partition:

* n
this will create the new partion

* press enter
if you want fdisk to give a default partion number

* press enter
this will create label on the partion

* +50G
this is mnt partition size if you are more into gaming or videos editings then you can add more

for home partition:

* n this will create the new partion

* press enter
if you want fdisk to give a default partion number

* press enter
this will create label on the partion

* press enter
home partition , this will use the rest all space left in your disk

for writing the partitions to disk:

* w to finilise the partitions

making the partitions:

! Select the partition number properly , i am using myne so replace those with yours

* mkfs.fat -F32 /dev/sda1
this will make the partition for ext4

* mkswap /dev/sda2
this will make the partition to the swap

* mkfs.ext4 /dev/sda3
this will make the partition to the file syatem

* mkfs.ext4 /dev/sda4
this will make the partition to the file system

mounting the partitions:

* mount /dev/sda3 /mnt
this will mount the partition to mnt

* mkdir /mnt/home
this will make the home directory

* mount /dev/sda4 /mnt/home
this will mount the partition to home

* swapon /dev/sda2
this will make the swap

for fstab:

* mkdir /mnt/etc
this will make etc directory

* genfstab -U -p/mnt >> /mnt/etc/fstab
this will generate the fstab to the created directory

* cat /mnt/etc/fstab
this will show the partitions in fstab

connecting to network and setting up mirrorlist:

* iwctl
interface prompt

*device list
to show the available devices

*station wlan0 scan
mine is wlan0 replace it according to yours

*station wlan0 get-networks
will display the available wifi names

*station wlan0 connect wifiname
replace the wifiname with yours, it will ask the passphrase

*exit
this will exit the interface

* reflector --country india --save /etc/pacman.d/mirrorlist
will save the mirrors to the mirrorlist mirrors

reflector is preinstalled in the iso, so change the country according to yours

* pacman -Syy
this will sync the repositories

! if the net speed is low change the mirrors and try again

installing the kernel and drivers for the system:

* pacstrap -i /mnt base base-devel linux-firmware
this will start installing the base into the mnt directory

* arch-chroot /mnt
this will mount the mnt directory

! here we have two types of kernels normal and long-term choose one according to your need

* pacman -S linux linux-headers nano
this for normal linux kernel

* pacman -S linux-lts linux-lts-headers nano
this is for long term linux kernel

* pacman -S networkmanager wpa_supplicant wireless_tools dialog network-manager-applet
this will install the required tools for network

* systemctl enable NetworkManager
this will enable the network manager on booting

! change the kernel according to your installed kernel

* mkinitcpio -p linux
bashscript used to create an initial ramdisk environment

* nano /etc/locale.gen
this will list the locale list

remove the "#" in front of the your prefered locale and save it

if you dont know about the locale use the en_us-utf8, bcz most of the apps use it

* locale-gen
this will generate the locale for your kernel

* passwd
this for setting the root password

* useradd -m -g users -G wheel username
this will add a user to the wheel group

* passwd username
setting password for the user

* EDITOR=nano visudo
this will edit the sudo config

remove the "#" in front of %wheelall=(all)all

* pacman -S grub efibootmgr dosfstools os-prober mtools
installing grub and required tools for booting

* mkdir -p /boot/EFI
this will make boot directory

* mount /dev/sda1 /boot/EFI
this will make the partition to Boot

! please make sure no errors after installing if it shows erros search online for other grub installation

* grub-install --target=x86_64-efi --bootloader-id=GRUB --efi-directory=/boot/EFI --no-nvram --removable
this will install grub to the boot directory

* mkdir /boot/grub/locale
this will make the locale directory

* cp /usr/share/locale/en\@quot/LC_MESSAGES/grub.mo /boot/grub/locale/en.mo
this will copy the grub configs

* grub-mkconfig -o /boot/grub/grub.cfg
setting up the grub config file so that it will make the arch boot

* pacman -S intel-ucode xorg-server mesa
this will drivers for intel graphics

* pacman -S nvidia xorg-server
for nvidia graphics

* exit

* reboot

After rebooting Arch linux will greet you with an terminal

To make Arch usable as other distros we have to download and start the desktop, if not you just keep stuck with the terminal

Here we have two types of environments:-

1. Desktop-environment

2. Window-manager

! There are lot of desktop managers & window managers, but I am giving very few of those

Login as the user which have sudo preferences

! Click Me for Options

click on headings for more

1. Desktop-environment:

I. KDE Plasma:

* sudo pacman -S sddm plasma konsole dolphin firefox
this installs the kde and required apps for normal usage

* sudo systemctl enable sddm
we enable the desktop manager to load on boot

* sudo systemctl start sddm
this will start the desktop manager

II. Gnome:

* sudo pacman -S gnome firefox
this will install the gnome and browser

* sudo systemctl enable gdm
this will enable the desktop manager to load on boot

* sudo systemctl start gdm
this will start the desktop manager

III. Xfce:

* sudo pacman -S xfce4 xfce4-goodies lxdm
this will install xfce

* sudo systemctl enable lxdm
this will enable the disktop manager to load on boot

* sudo systemctl start lxdm
this will start the desktop manager

IV. Lxde:

* sudo pacman -S lxde lxdm
this for installing the lxde

* sudo systemctl enable lxdm
this will enable the desktop manager to load on boot

* sudo systemctl start lxdm
this will start the desktop manager

2. Window-manager:

For window managers we need config files for basic operation so search online for config files

I. i3wm:

* sudo pacman -S i3 terminator thunar firefox xorg-xinit
installing the i3 and other required apps

* touch .xinitrc
we have to create the config file so that it can load the window manager

* nano .xinitrc
we are editing the config file

Add exec i3 to the end of the config, so that we can run the window manger

* startx
this will start the window manager

Arch user repository(AUR)

The Arch User Repository (AUR) is a community-driven repository for Arch users. It contains package descriptions (PKGBUILDs) that allow you to compile a package from source with makepkg and then install it via pacman.

by this we can install few of the applications by using, yay -S

yay:

* sudo pacman -S git
we need git package to clone the files into the required directory

* git clone https://aur.archlinux.org/yay.git
this will clone the git package into the home directory

* cd yay
this will change the directory to yay

* makepkg -si
this will install the yay package

* yay -S pacmac-aur
this will install the software manager
